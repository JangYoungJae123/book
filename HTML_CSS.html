1.1 Visual Studio Code

핵심 개념

Visual Studio Code는 마이크로소프트에서 개발한 무료 소스 코드 편집기로, 웹 개발에 필수적인 도구입니다.

정의

가볍고 빠르며 풍부한 확장 기능을 가진 통합 개발 환경(IDE) 역할을 하는 소스 코드 편집기입니다.

설치

Visual Studio Code 공식 웹사이트에서 다운로드하여 설치할 수 있습니다.

필수 확장 프로그램

  • Live Server: 웹 페이지를 실시간으로 브라우저에 표시하여 개발 효율성을 높입니다.
  • Prettier - Code formatter: 코드 서식을 자동으로 지정하여 일관된 코드 스타일을 유지합니다.

유용한 단축키

탐색창 열기/닫기, 다른 이름으로 저장, 선택 줄 복사/이동/삭제, 다중 라인 수정 등 다양한 단축키를 활용할 수 있습니다.

  • command(Ctrl) + B: VSC 왼쪽 탐색창 열기/닫기
  • command(Ctrl) + shift + S: 다른 이름으로 저장
  • shift + option(Alt) + (위, 아래): 선택 줄 복사
  • option(Alt) + 위아래 방향키: 선택 줄 이동
  • command(Ctrl) + shift + k: 한 줄 삭제
  • command(Ctrl) + option + (위, 아래): 다중 라인 수정

실습: VS Code 설치하기

1.2 수업 전 알아 두기

웹 브라우저

Chrome 사용을 권장합니다. 사용자 수가 가장 많고 개발자 도구가 가장 잘 되어 있습니다.

확장 프로그램 (Chrome)

  • HeadingsMap: 웹 페이지의 제목 태그들을 목차처럼 볼 수 있게 해줍니다.
  • OpenWAX: 웹 페이지의 접근성을 진단하는 데 유용합니다.

폴더 및 파일 이름 규칙

  • 공백 없이 영문 소문자를 사용합니다. 이는 컴퓨터와 웹 서버가 대소문자를 구분하고, 공백 처리가 일관적이지 않아 발생할 수 있는 오류를 줄이기 위함입니다.
  • 언더바(_) 대신 하이픈(-)을 사용하는 것이 좋습니다. Google 검색 엔진은 하이픈을 단어 구분자로 인식하여 SEO에 도움이 됩니다 (예: lion-resting-on-the-road.png).

파일 경로

  • 상대경로: 현재 파일을 기준으로 하는 상대적인 경로입니다 (예: ./, ../, ./폴더명/).
  • 절대경로: http://, https://로 시작하는 인터넷상의 고유한 경로(URL)를 의미합니다.

개발자 도구

Chrome에서 F12 키를 눌러 열 수 있습니다. 웹 페이지의 HTML, CSS, JavaScript를 확인하고 수정하며, 네트워크, 콘솔 등의 탭을 통해 웹 페이지의 동작을 분석할 수 있습니다.

1.3 웹 Web 이란?

핵심 개념

웹(Web)은 인터넷을 기반으로 하는 수많은 서비스 중 하나로, HTTP 프로토콜을 사용하여 정보를 전송합니다.

World Wide Web (WWW)

전 세계적으로 분산된 수많은 문서와 리소스들이 서로 연결된 거대한 정보 네트워크입니다.

웹 페이지

일반적으로 HTML이라는 마크업 언어로 작성된 문서 형식으로 제공됩니다.

웹의 기초 구성 요소

  • HTML: 웹 콘텐츠의 의미와 구조를 정의합니다.
  • CSS: 웹 페이지의 모양과 표현을 담당합니다.
  • JavaScript: 웹 페이지의 기능과 동작을 담당합니다.

웹의 역사

  • 팀 버너스 리가 1990년에 초창기 웹 브라우저를, 1991년에 HTML을 개발했습니다. 초창기에는 약 20개의 태그만 지원하는 단순한 형태였습니다.
  • 1996년에 CSS가 등장하여 HTML을 꾸며주는 역할을 시작했습니다.
  • 같은 해 JavaScript(초기 이름 Mocha)가 넷스케이프 네비게이터2에 탑재되며 발전했습니다.

인터넷(Web !== Internet)

  • 인터넷: 전 세계 컴퓨터와 기기들을 연결하는 거대한 네트워크(인프라)입니다.
  • 웹: 인터넷을 기반으로 한 수많은 서비스 중 하나입니다.
  • HTTP (HyperText Transfer Protocol): 웹은 이 프로토콜을 사용하여 정보를 전송하며, 이는 컴퓨터나 장치 간 데이터를 주고받는 전송 규칙을 의미합니다.

인터넷 위 다른 서비스

웹 외에도 이메일(SMTP/POP3/IMAP), FTP(파일 전송), SSH(원격 제어) 등 다양한 서비스들이 인터넷 위에 존재하며 각기 다른 프로토콜을 사용합니다.

1장 요약

  • VS Code는 가장 인기 있는 웹 개발 도구 중 하나입니다.
  • 웹은 인터넷 위의 서비스 중 하나이며, HTTP 프로토콜을 사용합니다.
  • HTML5는 Living Standard로 지속적으로 개선되고 있습니다.
  • 파일 이름과 경로 규칙을 지켜야 합니다.

프론트엔드와 백엔드

  • 프론트엔드 개발자: 사용자 인터페이스(UI)를 다루며, HTML, CSS, JavaScript를 주력 언어로 사용합니다. React, Vue 등은 JavaScript 기반입니다.
  • 백엔드 개발자: 사용자에게 보이지 않는 서버와 데이터 처리를 다룹니다. Python, Java, C 등 다양한 프로그래밍 언어와 웹 프레임워크를 사용하며, 프론트엔드의 동작 방식을 이해하는 것이 중요합니다.

1.4 HTML5

정의

최신 웹 기술을 가리키는 용어로, 이전 버전인 HTML 4.01에 비해 다양한 새로운 기능과 요소를 포함합니다.

Living Standard

2019년부터 WHATWG와 W3C는 Living Standard를 HTML의 유일한 표준으로 발표했습니다. 이는 버전 번호 없이 HTML 표준을 지속적으로 업데이트하고 발전시키는 방식을 의미합니다.

<!DOCTYPE html>

이 선언문은 HTML Living Standard로 작성된 문서임을 브라우저에 알려주어, 페이지를 표준 모드로 렌더링하게 합니다. 과거의 복잡한 DTD(문서 유형 정의)와는 달리 간결해졌습니다.

2장 HTML

2.1 HTML 문서해부

index.html

웹 개발에서 웹사이트나 웹 애플리케이션의 진입점 또는 시작 페이지로 사용되는 특별한 파일명입니다. 웹 서버의 기본 설정과 개발 커뮤니티의 관례 때문입니다.

핵심 개념

<!DOCTYPE html>은 HTML5 문서를 선언하는 필수 태그로, 모든 HTML 파일의 첫 번째 줄에 작성해야 합니다.

<!DOCTYPE html>

HTML Living Standard 문서임을 선언하며, 브라우저가 완전 표준 모드로 렌더링하도록 합니다.

<html>

HTML 문서의 루트(최상위) 태그입니다. 모든 HTML 요소는 이 태그 내에 포함되어야 하며, lang 속성으로 페이지의 주 언어를 설정합니다 (예: lang="ko-KR").

<head>

문자 인코딩, 타이틀, 설명, 스타일시트, 스크립트 등 기계가 식별할 수 있는 문서 정보(메타데이터)를 담습니다.

주요 메타 태그들
  • <meta charset="utf-8">: 문자 깨짐 오류를 방지하고 전 세계 언어를 지원하는 문자 코드를 설정합니다.
  • <meta http-equiv="X-UA-Compatible" content="IE=edge">: Internet Explorer 브라우저에서 최신 표준 모드로 페이지를 보여주도록 호환성을 위해 작성합니다.
  • <meta name="viewport" content="width=device-width, initial-scale=1.0">: 모바일 장치 등 다양한 디바이스에서 웹 페이지의 표시 방식을 제어합니다.
  • <title>: 브라우저의 제목 표시줄이나 페이지 탭에 보이는 문서 제목을 정의합니다. 검색 엔진이 페이지를 선택하는 중요한 정보이며, 약 60자를 넘지 않도록 하는 것이 좋습니다.
  • <link>: 현재 문서와 외부 리소스(스타일 시트, 폰트, 파비콘 등)의 관계를 명시하고 연결합니다.

<body>

사용자에게 실제로 보이는 영역으로, 웹 페이지의 모든 콘텐츠가 이곳에 작성됩니다.

2.2 HTML 기본 문법

요소(Elements)

HTML을 구성하는 가장 기본적인 단위입니다.

태그(Tag)

<>로 시작하는 시작 태그(여는 태그)와 </>로 끝나는 종료 태그(닫는 태그)의 쌍으로 구성되며, 웹을 구성하는 각 요소의 역할을 정의합니다.

빈 요소

닫는 태그가 없는 요소들을 의미합니다 (예: <br>, <img>, <input>). 현대 HTML에서는 /를 넣어서 닫아주지 않아도 됩니다.

주석

<!-- 주석 --> 형태로 작성하며, 프로그램 동작에 영향을 미치지 않습니다. 코드 설명을 달거나 특정 소스를 임시로 비활성화("주석 처리")하는 용도로 사용됩니다.

가독성을 위해 최소한으로 남기고, 시맨틱 태그와 직관적인 클래스명으로 코드를 이해하기 쉽게 작성하는 것이 좋습니다.

태그의 관계

  • 부모(Parent) 요소: 다른 요소를 포함하는 요소입니다 (예: <section>은 <h1>의 부모).
  • 자식(Child) 요소: 부모 요소 내부에 직접 포함된 요소입니다 (예: <h1>은 <section>의 자식).
  • 형제(Sibling) 요소: 같은 부모 요소를 가지는 요소들입니다.
  • 자손(Descendant) 요소: 어떤 요소의 내부에 포함된 모든 요소를 말하며, 자식 요소뿐만 아니라 자식의 자식 요소까지 포함합니다.
요소 간 관계의 중요성

CSS 선택자를 사용하여 특정 요소를 선택하고 스타일을 적용할 때, 이러한 관계를 활용하여 더 정확하고 효율적으로 스타일을 지정할 수 있습니다 (예: section p { color: blue; }).

핵심 개념

텍스트 의미론적 마크업은 단순한 스타일링이 아닌 내용의 의미를 HTML로 표현하는 것입니다. 이는 접근성과 SEO에 핵심적인 역할을 합니다.

3.1 텍스트와 관련된 태그들

<h> (제목 태그)

<h1>부터 <h6>까지 있으며, 웹 페이지의 논리 구조를 정의하고 중요한 내용을 나타냅니다. 숫자가 작을수록 크고 중요한 제목을 의미하며, 검색 엔진 최적화(SEO)와 스크린 리더 사용자에게 매우 중요합니다.

  • <h1> 태그는 한 페이지에 한 번만 사용하는 것을 권장합니다 (예: 웹사이트 로고).
  • 스타일 목적으로 h 태그를 사용하지 않고, 항상 순서대로 작성해야 합니다.

<a> (하이퍼링크)

href 속성을 통해 연결할 URL을 지정하여 웹 페이지 간 이동을 가능하게 합니다.

주요 속성
  • href 속성값: 절대 URL, 상대 URL, 같은 페이지 내 요소 (#section1), tel:(전화 연결), mailto:(이메일 연결) 등
  • target 속성: 링크가 열리는 위치를 지정하며, _blank(새 탭)가 가장 많이 사용됩니다.
  • download 속성: 링크 클릭 시 페이지 이동 대신 파일 다운로드를 수행합니다.

<a> 태그는 인라인 요소이지만, 예외적으로 블록 요소들을 감쌀 수 있습니다.

<p> (문단)

paragraph의 약어로, 하나의 문단을 나타냅니다. 문단 사이에 자동으로 여백이 생기고, 검색 엔진이 내용을 더 잘 이해할 수 있도록 돕는 블록 요소입니다.

<br> (줄 바꿈)

break의 약어로, 줄을 나눕니다. 닫는 태그가 없으며, 오직 줄 바꿈 기능만 수행합니다. 여백을 위해 여러 번 사용하지 않도록 권장됩니다.

<wbr>

브라우저가 줄을 바꿀 수 있는 위치를 지정하는 방법입니다.

  • 한글의 경우는 CSS의 word-break:keep-all 속성과 함께 사용

강조 및 의미 태그들

<strong>

중대하거나 긴급한 콘텐츠를 나타내는 태그입니다. 기본적으로 굵은 글꼴이 적용되며, 스크린 리더는 거센 억양으로 발음하여 중요 정보를 강조합니다.

<em>

Emphasis의 약어로, 텍스트의 강세나 강조를 나타냅니다. 기울임 글꼴이 적용되며, 음성으로 읽을 때도 의미를 강조하여 전달합니다. 중첩하여 더 큰 강세를 나타낼 수 있습니다.

주의사항

<b>와 <i>는 과거에 스타일 목적으로 사용되었으나, 의미론적이지 않으므로 사용하지 않는 것을 권장합니다. strong, em 등은 모두 스타일이 아닌 의미론적인 목적으로 사용해야 하며, 스타일은 CSS를 이용해야 합니다.

인용 관련 태그들

<q>

짧은 인용문에 사용되며, 브라우저가 자동으로 인용 부호를 추가합니다. cite 속성으로 출처 URL을 지정할 수 있습니다.

<blockquote>

긴 인용문을 나타냅니다.

<cite>

저작물의 출처를 표기할 때 사용하며, 제목을 반드시 포함해야 합니다.

기타 유용한 태그들

  • <code>: 짧은 코드 조각이나 한 줄의 코드를 나타낼 때 사용
  • <pre>: HTML에 작성한 내용 그대로(공백 유지)를 표현
  • <mark>: 형광펜으로 칠한 것처럼 표시되는 하이라이트 텍스트
  • <address>: 해당 콘텐츠에 대한 연락처 정보를 나타냄
  • <abbr>: 줄임말이나 약어임을 표시할 때 사용
  • <sup>: 위 첨자 요소
  • <sub>: 아래 첨자 요소
  • <kbd>: 키보드 입력을 나타냄

3.2 목록 관련 태그 ul,ol,dl

<ol> (ordered list)

순서가 있는 목록을 나타냅니다. 숫자나 문자를 이용해 항목이 정렬된 목록을 만들 때 사용되며, type, start, reversed 등의 속성을 가질 수 있습니다 (예: 레시피 단계, 브레드크럼).

<ul> (unordered list)

unordered list의 약자로, 순서가 중요하지 않은 비순차적 목록을 의미합니다 (일반적으로 불릿으로 표현).

ol vs ul

항목의 순서를 바꿨을 때 그 의미도 바뀐다면 <ol>, 그렇지 않고 순서가 중요하지 않다면 <ul>을 사용합니다.

<li> (list item)

목록의 항목을 나타냅니다. 단독으로 사용할 수 없으며, <ol> 또는 <ul>의 자식 요소로만 사용 가능합니다.

<dl> (description list)

용어와 그 설명을 짝지어 표현하는 설명 목록을 나타냅니다. 사전처럼 체계적으로 정보를 정리할 때 유용합니다.

구성 요소
  • <dl>: 전체 목록 컨테이너
  • <dt>: 정의될 용어
  • <dd>: 용어에 대한 설명

실습: 의미론적 마크업 연습

  1. 뉴스 기사를 작성하되, 제목, 부제목, 본문, 인용문을 적절한 태그로 마크업해보세요.
  2. 강조가 필요한 부분은 <strong>과 <em> 태그를 사용해보세요.
  3. 순서가 있는 레시피와 순서가 없는 재료 목록을 각각 작성해보세요.
  4. 용어 사전을 <dl>, <dt>, <dd> 태그로 작성해보세요.

4장 어떤 태그는 줄바꿈이 되고 어떤 것은 안될까?

4.1 블록 레벨 요소 vs 인라인 요소

핵심 개념

블록 레벨 요소는 새로운 줄에서 시작하여 전체 너비를 차지하며, 인라인 요소는 텍스트 흐름에 따라 배치되고 콘텐츠만큼의 공간만 차지합니다.

참고: 블록 레벨과 인라인 요소의 분류는 HTML 4.01까지 사용되었으며, 현재 HTML 표준에서는 콘텐츠 카테고리 집합으로 대체되었습니다.

<div> (블록 컨테이너)

division의 약자로, 웹 페이지의 레이아웃을 만들고 콘텐츠를 분할하는 블록 레벨 컨테이너입니다. 영역을 구분 짓거나 무리를 지을 때 사용됩니다.

스타일 적용을 위한 용도로 사용하는 것을 권장하며, SEO, 코드 가독성, 접근성을 위해 시맨틱 태그를 우선적으로 사용해야 합니다.

<span> (인라인 컨테이너)

인라인 컨테이너로, 스타일을 적용하거나 인라인 요소를 묶는 데 사용됩니다. 본질적으로는 아무런 의미를 나타내지 않습니다.

블록 요소 (Block-level elements)

  • 부모 요소의 전체 공간을 차지하며 새로운 줄에서 시작하고, 가능한 모든 너비를 차지합니다.
  • 이전/이후 요소 사이에 줄 바꿈이 일어납니다.
  • 페이지의 구조적 요소를 나타낼 때 사용됩니다.
  • 인라인 요소를 포함할 수 있지만, 인라인 요소 안에 중첩될 수는 없습니다 (예외: <a> 태그는 블록 요소를 중첩할 수 있습니다).
  • width, height 크기를 지정할 수 있으며, padding, border, margin 속성 사용이 가능합니다.
대표적인 블록 요소

<div>, <p>, <h1>~<h6>, <header>, <footer>, <section>, <article>, <nav> 등

인라인 요소 (Inline elements)

  • 항상 블록 레벨 요소 내에 포함되며, 콘텐츠의 흐름을 끊지 않고 콘텐츠에 따라 할당된 공간만 차지합니다.
  • 새로운 줄을 만들지 않습니다.
  • width, height 크기를 지정할 수 없으며, margin 속성 중 상하 간격은 적용되지 않습니다 (padding, border는 적용 가능).
대표적인 인라인 요소

<a>, <em>, <strong> 등

인라인-블록 요소 (Inline-block elements)

  • 인라인 요소처럼 전후 줄 바꿈 없이 한 줄에 다른 요소들과 나란히 배치됩니다.
  • 블록 요소처럼 너비와 높이 지정 및 margin, padding 상하 간격 지정이 가능합니다.
대표적인 요소

button, input, select

4.2 콘텐츠 카테고리 (HTML5)

플로우 콘텐츠 (Flow Content)

대부분의 HTML 요소를 포함하며, 웹 문서에서 일반적으로 사용하는 콘텐츠입니다 (예: <div>, <p>, <a>, <table>, <header>, <article>).

대화형 콘텐츠 (Interactive Content)

사용자가 상호작용할 수 있는 요소들입니다 (예: <a>, <button>, <input>, <select>, <details>, <label>).

구획 콘텐츠 (Sectioning Content)

문서의 구조를 정의하고 섹션을 나누는 데 사용됩니다 (예: <article>, <aside>, <nav>, <section>, <header>, <footer>).

제목 콘텐츠 (Heading Content)

섹션의 제목을 정의합니다 (예: <h1>~<h6>).

구문 콘텐츠 (Phrasing Content)

텍스트와 인라인 요소를 포함하며, 텍스트의 의미와 구조를 정의합니다 (예: <span>, <a>, <em>, <strong>, <img>, <code>, <time>).

내장 콘텐츠 (Embedded Content)

외부 소스의 콘텐츠를 포함합니다 (예: <img>, <audio>, <video>, <iframe>, <source>, <track>).

메타데이터 콘텐츠 (Metadata Content)

문서의 메타데이터를 정의하는 요소들입니다. 주로 <head> 요소 내에 위치합니다 (예: <base>, <link>, <meta>, <title>).

5.1 이미지 img

<img> 태그

HTML 문서에 이미지를 삽입하는 데 사용됩니다. 과거에는 이미지를 보려면 링크를 클릭해야 했지만, <img> 태그 덕분에 글자와 이미지를 한 번에 볼 수 있게 되었습니다.

주요 속성

src 속성

이미지 파일의 경로를 지정합니다.

alt 속성 (대체 텍스트)

src에 지정한 이미지를 제대로 불러오지 못했을 때 이미지 대신 표시되는 텍스트입니다. 시각장애인을 위한 스크린 리더를 지원하며, 이미지가 단순히 페이지를 꾸미는 용도라면 alt=""와 같이 빈 값으로 둡니다.

title 속성

이미지에 대한 추가 정보를 제공하는 텍스트로, 마우스를 이미지 위에 올리면 툴팁으로 표시됩니다. alt 속성을 대체할 수 없으며, alt와 동일하거나 부연 설명으로 사용하지 않는 것이 좋습니다.

이미지 파일명

Google을 포함한 대부분의 검색 엔진은 이미지 검색 순위 결정 시 이미지 파일명을 중요한 요소로 활용하므로, 이미지 내용을 설명하는 파일 이름 (예: flower.jpg 대신 img234235.png)을 사용하는 것이 좋습니다.

관련 태그들

<map>, <area> 태그
  • <map>: 이미지 맵을 정의하는 태그입니다.
  • <area>: 이미지 맵 내에서 클릭 가능한 영역을 정의하는 태그입니다.
<figure>, <figcaption> 태그
  • <figure>: 문서의 주요 흐름과 독립적으로 참조될 수 있는 자체 완결적인 콘텐츠 블록(일러스트레이션, 다이어그램, 사진, 코드 목록 등)을 나타냅니다.
  • <figcaption>: <figure> 요소에 대한 캡션(설명)을 추가할 때 사용합니다.

6.1 웹 접근성

정의

어떠한 사용자(장애인, 고령자 등)이던지 웹에서 제공하는 모든 정보에 접근할 수 있도록 보장하는 것입니다.

웹의 힘

웹의 창시자 팀 버너스 리 경은 "웹의 힘은 그것의 보편성에 있다. 장애에 구애 없이 모든 사람이 접근할 수 있는 것이 필수적인 요소이다."라고 강조했습니다.

중요성

  • 약 13억 명(전 세계 6명 중 1명)이 심각한 장애를 경험합니다.
  • 장애로 인한 건강 불평등, 낙인, 차별 등을 해소하기 위한 노력의 일환입니다.
  • 법적 근거: 장애인차별금지법에 따라 웹 접근성을 준수해야 하며, 위반 시 시정 권고, 과태료, 민/형사 고발이 가능합니다.

정보에 접근하는 다양한 방법

  • 시각 장애 (전맹): 스크린 리더, 점자 정보 단말기
  • 시각 장애 (저시력): 화면 확대, 색 전환(흑백/고대비)
  • 신체적 장애: 헤드 포인터, 빅키 키보드, 트랙볼 마우스 등 특수 보조 기기

WCAG (Web Content Accessibility Guidelines)

W3C에서 제시한 웹 콘텐츠 접근성 가이드라인 표준으로, 장애가 있는 사용자가 웹을 더 쉽게 이용할 수 있도록 준수해야 하는 지침입니다.

4가지 핵심 원칙
  1. 인식 가능(Perceivable): 모든 사용자가 서비스 콘텐츠를 인식할 수 있어야 합니다.
  2. 운용 가능(Operable): 모든 사용자가 서비스의 기능을 운용할 수 있어야 합니다 (예: 마우스 없이도 이용 가능).
  3. 이해 가능(Understandable): 모든 사용자가 서비스의 콘텐츠, 기능 사용법 등을 이해하기 쉬워야 합니다.
  4. 견고(Robust): 사용자가 이용하는 모든 기기 및 브라우저에서 접근, 사용 가능해야 합니다.
WCAG 규정 준수 수준
  • A 레벨 (최소 준수): 웹사이트 접근성을 본질적으로 불가능하게 만드는 요소를 금지
  • AA 레벨 (허용 가능한 보편적 규정 준수): 전 세계 대부분의 접근성 규칙 및 규정에서 사용되며, 대다수의 장애인이 사용하고 이해할 수 있음
  • AAA 레벨 (최적의 규정 준수): 거의 모든 사용자가 사이트의 모든 기능을 경험하는 이상적인 수준

간단한 접근성 확인법

  • 탭(Tab) 버튼으로 탐색: 모든 버튼, 앵커, 인풋 요소가 탭으로 탐색 가능한지 확인
  • 포커싱 확인: 포커싱된 요소에 명확한 시각적 표시가 있어야 합니다
  • 호버 스타일 확인: 색상 변화만으로 호버를 표현하지 말고, 밑줄이나 테두리 스타일 등 직접적인 변화를 주어 저시력/색맹 사용자를 배려
  • 접근성 관련 툴 사용: Lighthouse, Siteimprove Accessibility Checker 등

웹 접근성을 위한 HTML 작성 원칙

1. 시맨틱(Semantic) 태그 사용하기

HTML5의 시맨틱 태그는 각 요소의 의미를 명확히 합니다. 이는 스크린 리더가 웹 페이지의 구조를 이해하고 사용자에게 내용을 효과적으로 전달하는 데 필수적입니다.

  • 잘못된 예시:

    <div class="header"></div>
<div class="nav"></div>
<div class="content"></div>

  • 올바른 예시:

    <header></header>
<nav></nav>
<main></main>
<footer></footer>

    <header>, <nav>, <main>, <footer> 등의 태그는 그 자체로 의미를 담고 있어 스크린 리더가 '여기는 웹 페이지의 주요 콘텐츠 영역이구나'라고 인식할 수 있게 해줍니다.

2. 이미지에 대체 텍스트(alt) 제공하기

<img> 태그에는 반드시 alt 속성을 포함하여 이미지를 보지 못하는 사용자에게 이미지의 내용을 설명해야 합니다.

  • 정보를 제공하는 이미지: 이미지의 내용을 구체적으로 설명합니다.

    <img src="flower.jpg" alt="푸른 하늘 아래 핀 붉은 장미">

  • 장식용 이미지: 내용 전달에 필요 없는 이미지는 alt=""로 비워두면 스크린 리더가 이를 건너뜁니다.

    <img src="divider.png" alt="">

3. 폼 컨트롤에 레이블(label) 연결하기

<input>, <textarea> 같은 폼 요소는 <label> 태그와 연결해야 합니다. 이는 스크린 리더가 폼의 용도를 사용자에게 정확히 알려주고, 사용자가 레이블을 클릭하여 입력 칸을 활성화할 수 있게 합니다.

  • 잘못된 예시:

    아이디
<input type="text">

  • 올바른 예시:

    <label for="username">아이디</label>
<input type="text" id="username">

    for 속성과 id 속성을 연결하여 두 요소가 서로 관련 있음을 명시합니다.

4. 제목 태그(h1~h6)를 구조적으로 사용하기

제목 태그는 콘텐츠의 계층을 나타내야 합니다. <h1>은 페이지의 가장 중요한 제목에 단 한 번만 사용하고, 그 외의 제목은 순서대로 <h2>, <h3> 등을 사용해야 합니다. 이는 책의 목차와 같이 웹 페이지의 논리적 흐름을 구성합니다.

  • 잘못된 예시:

    <h1>메인 제목</h1>
<h4>소제목</h4>
<h2>다른 소제목</h2>

  • 올바른 예시:

    <h1>메인 제목</h1>
<h2>대분류</h2>
<h3>중분류</h3>
<h4>소분류</h4>

5. 링크와 버튼을 명확하게 구분하기

  • 링크(<a>): 페이지 이동 등 탐색의 용도로 사용합니다. 링크 텍스트는 '여기를 클릭' 대신 '자세한 내용 보기', '문의하기'와 같이 목적을 명확히 설명해야 합니다.

  • 버튼(<button>): 폼 제출, 팝업 열기 등 특정 동작을 수행할 때 사용합니다.

6. 문서의 언어 설정하기

HTML 문서의 언어를 <html> 태그에 lang 속성으로 지정하면, 스크린 리더가 해당 언어에 맞춰 발음하여 사용자에게 정확한 정보를 전달할 수 있습니다.

<html lang="ko">
...
</html>

6.2 Layout History

  1. 단순한 텍스트의 나열: 웹 초창기에는 레이아웃 도구가 없어 텍스트 정렬만 가능했습니다.
  2. 테이블로 레이아웃 구성: <table> 태그를 표 데이터 본래의 목적과 다르게 웹 페이지 배치를 위해 사용했습니다. (현재도 뉴스레터 제작 등에서 사용)
  3. 프레임으로 레이아웃 구성: <iframe> 태그를 사용하여 여러 HTML 페이지를 한 페이지에 결합하여 레이아웃을 만들었습니다.
  4. <div> 사용: <div> 태그와 CSS의 Box Model, float, position 속성 등을 이용하여 레이아웃을 만들던 시기입니다. 여러 브라우저별 표준이 정의되지 않아 다양한 기법이 난무했습니다.
  5. 시맨틱 마크업, Flex, Grid: 다양한 웹 기술 발전, 검색 엔진 최적화(SEO), 접근성, 반응형 웹 등의 이유로 생겨난 최신 레이아웃 기법입니다.

시맨틱 마크업의 중요성

  • SEO (검색 엔진 최적화): 검색 엔진이 페이지의 콘텐츠를 더 잘 이해하게 돕습니다
  • 장애가 있는 사용자: 페이지 탐색 시 푯말 역할로 웹 접근성을 향상시킵니다
  • 가독성 및 유지보수 용이: 태그의 의미가 명확하여 코드를 읽고 관리하기 쉽습니다

6.3 Semantic Elements (시맨틱 요소)

목적

인터넷 정보들을 컴퓨터가 이해하고 가공할 수 있도록 하고, 웹 접근성을 향상시켜 정보 격차를 해소하는 것이 목적입니다.

<header>

소개 및 탐색에 도움을 주는 콘텐츠를 담습니다 (회사명, 제목, 로고, 검색 폼, 작성자 이름 등). HTML 문서가 여러 섹션으로 구성될 때 각 섹션의 시작 부분에 사용할 수 있습니다.

  • <head> 태그와 혼동하지 않도록 주의해야 합니다
  • 헤더를 중첩 사용하거나 헤더 안에 푸터를 사용할 수 없습니다

<nav>

navigation bar의 약자로, 현재 페이지 내 또는 다른 페이지로의 링크를 보여줍니다. 내부에는 메뉴, 목차, 브레드크럼(Breadcrumb) 등이 포함됩니다. 하나의 웹 페이지에 여러 개의 <nav> 태그를 가질 수 있습니다.

<footer>

페이지의 작성자, 저작권 정보, 페이지 관련 문서 등의 내용을 담습니다.

<main>

<body>의 주요 콘텐츠를 나타냅니다. 웹 페이지에서 한 번만 사용할 수 있으며, 사이드바, 탐색 링크 등 반복되는 콘텐츠를 포함해서는 안 됩니다 (검색 폼이 주요 기능이라면 예외).

<article>

독립적으로 구분하여 배포하거나 재사용할 수 있는 콘텐츠 블록을 나타냅니다 (블로그 포스트, 뉴스 기사, 사용자 댓글, 위젯 등). 사이트의 다른 기능에 영향을 주지 않고 독립적인 기능을 수행하며, 제목 요소를 자식으로 포함해야 합니다.

<section>

문서 내에서 특정한 섹션을 정의하는 데 사용됩니다. 일반적으로 주제나 콘텐츠를 구분하기 위해 사용되며, 제목 요소를 자식으로 포함해야 합니다 (예: 뉴스의 사회면 / 연예면).

<article> vs <section>

독립적으로 사용 가능하다면 <article>을 우선 고려하고, 웹 페이지의 앞뒤 문맥에 연결성이 필요하거나 더 적합한 의미 요소가 없을 때 <section>을 사용합니다. 단순 스타일링 목적이라면 <div>를 사용합니다.

<aside>

문서의 주요 내용과 간접적으로 연관된 부분을 나타냅니다. 문서의 주요 흐름을 따라가지 않고, 보조적인 역할만 하는 공간입니다 (예: 광고 배너, 사이드바).

<hr>

이야기에서 장면 전환 혹은 문단 안에서 주제가 변경되었을 때 사용합니다. 단락을 구분할 때 사용하므로 <p> 태그 내의 사용은 지양합니다.

학습 목표

  • HTML 테이블의 구조와 접근성을 고려한 표 작성법 학습
  • 비디오, 오디오 등 멀티미디어 콘텐츠 삽입과 제어 방법 이해
  • iframe을 통한 외부 콘텐츠 포함 기법과 주의사항 파악
  • SVG와 Canvas의 차이점 및 활용 방법 학습
  • 사용자 입력을 받는 다양한 폼 요소들의 활용법 습득
  • 웹 표준을 준수한 다양한 HTML 요소들의 올바른 사용법 이해

7장 웹페이지에 다양한 컨텐츠를 넣고 싶어!

7.1 표(table)

<table>

테이블을 생성할 때 사용하며, 테이블 데이터의 컨테이너 요소입니다.

<tr>

table row의 약자로, 테이블의 행을 정의합니다.

<td>

table data의 약자로, 테이블 셀의 내용을 나타냅니다.

<th>

table header의 약자로, 테이블의 행 또는 열의 제목을 나타내는 셀입니다.

주요 속성
  • scope 속성: <th> 태그에 사용하여 헤더가 어떤 셀에 대한 제목 셀인지 명확하게 지정합니다 (col, row, colgroup, rowgroup 값)
  • colspan, rowspan 속성: 셀을 병합할 때 사용하며, 각각 열 병합과 행 병합을 의미합니다

추가 테이블 요소들

  • <caption>: 테이블의 제목이나 설명을 의미하며, <table>의 첫 번째 자식으로 사용
  • <thead>: 테이블의 머리글 구역
  • <tbody>: 테이블의 본문 구역
  • <tfoot>: 테이블의 바닥글 구역
  • <colgroup>과 <col>: 테이블 열 단위 스타일 지정

7.2 video & audio

<video>

웹 페이지에 동영상을 삽입할 때 사용합니다. 과거의 플래시 플레이어를 대체하여 발전했습니다.

주요 속성
  • src: 동영상 주소
  • controls: 재생/정지 조작 메뉴
  • autoplay: 자동 재생
  • muted: 음소거
  • loop: 반복 재생
  • poster: 재생 전 이미지
  • preload: 미리 로드 방식

<audio>

웹 페이지에 오디오 콘텐츠를 포함할 때 사용합니다. MP3, WAV, Ogg가 공식적으로 지원하는 표준 오디오 파일입니다.

<source>

<video> 또는 <audio> 태그 내에서 여러 미디어 파일을 지정하여, 브라우저가 지원하는 형식을 선택하여 재생할 수 있도록 합니다.

autoplay (자동 재생)는 사용자의 의도와 달리 갑작스러운 소리나 영상 재생으로 불쾌감을 줄 수 있으므로, 가능한 한 지양하는 것이 좋습니다.

7.3 iframe

<iframe> (inline frame)

현재 웹 페이지에 다른 HTML 페이지를 포함시켜 중첩하는 기능을 제공합니다.

주요 속성
  • src: 삽입될 문서 주소
  • width, height: 너비/높이
  • scrolling: 스크롤바 유무
  • allow: 허용할 기능
주의 사항
  • 일부 사이트에서는 보안상의 이유로 <iframe> 포함을 거부할 수 있습니다
  • 성능 향상을 위해서는 메인 콘텐츠가 완전히 로딩된 후 JavaScript로 src 속성을 설정하는 것이 좋습니다

7.4 SVG, Canvas

SVG (Scalable Vector Graphics)

XML 기반의 2차원 그래픽으로, 픽셀이 아닌 수학적 경로와 도형으로 이미지를 표현합니다. 어떤 크기로 확대해도 품질이 유지되는 확장 가능한 벡터 그래픽입니다.

장점
  • 확장성 (어떤 해상도에서도 선명)
  • CSS/JavaScript로 편집 가능
  • 접근성 (텍스트 정보 포함)
  • SEO 친화적
  • 단순한 도형의 경우 파일 크기가 작음
사용 방법
  • 외부 파일로 사용 (<img> 태그)
  • CSS background 배경으로 넣기
  • 인라인으로 넣기 (완전 제어 가능)

Canvas

7.5 양식(form)

form

사용자로부터 입력을 받기 위한 양식을 작성하는 태그들을 통틀어 form이라고 합니다. 입력한 데이터를 제출, 전송할 필요가 없다면 form 태그를 사용하지 않아도 됩니다.

주요 속성
  • method 속성: GET (URL에 데이터 첨부, 보안 취약) / POST (별도 첨부, 보안성 높음)
  • enctype 속성: application/x-www-form-urlencoded (기본) / multipart/form-data (파일 업로드)
  • action 속성: 양식 데이터를 처리할 프로그램의 URL
  • autocomplete 속성: 브라우저의 자동 완성 기능 제어 (on/off)

input

사용자로부터 다양한 유형의 데이터를 입력받는 양식 컨트롤입니다.

공통 속성
  • type: 컨트롤 유형
  • name: 컨트롤 이름
  • value: 컨트롤 값
  • placeholder: 입력 힌트
  • required: 필수 입력
  • disabled: 비활성화
  • readonly: 읽기 전용

label

사용자 인터페이스의 항목을 나타냅니다. input과 함께 사용하여 스크린 리더 사용자가 입력해야 하는 내용을 쉽게 이해할 수 있게 하며, label을 클릭하여 input에 포커스를 이동시킬 수 있습니다.

기타 form 요소들

  • <select>: 옵션 메뉴를 제공하는 드롭다운 목록
  • <option>: select 메뉴의 각 옵션을 정의
  • <optgroup>: option 요소들을 그룹으로 묶음
  • <fieldset>: 다양한 선택 항목을 그룹화
  • <legend>: fieldset의 제목을 제공
  • <datalist>: input 요소에 미리 정의된 옵션 목록 제공
  • <textarea>: 여러 줄의 텍스트를 입력받음
  • <button>: 사용자가 클릭하여 상호작용할 수 있는 요소

9.1 CSS 기본

정의

우리가 작성한 HTML의 스타일과 레이아웃 등을 꾸미는 언어입니다. HTML과 마찬가지로 프로그래밍 언어는 아니지만 웹 개발에서 중요한 역할을 합니다.

CSS 등장 배경

1990년대 초반, 초기의 HTML은 문서 구조 정의에만 사용되었고 스타일링 기능이 거의 없었습니다. 개발자들이 HTML 태그 내에 직접 스타일 속성을 삽입하는 방식은 코드를 복잡하게 만들고 유지보수를 어렵게 했습니다.

CSS의 탄생

1994년 하콘 비움 리가 스타일 시트 개념을 제안했고, 1996년 12월 W3C에 의해 최초의 CSS(Cascading Style Sheets) 표준으로 공식화되었습니다.

CSS의 장점

  • 구조와 스타일의 분리: HTML은 콘텐츠와 구조, CSS는 스타일과 레이아웃을 정의하여 문서의 구조적 의미를 명확히 하고 유지보수성을 높입니다
  • 일관된 디자인 유지: 여러 페이지에 동일한 스타일을 쉽게 적용할 수 있어 웹사이트 전체의 일관된 디자인을 유지합니다
  • 재사용성 및 효율성 향상: 스타일 시트를 한 번 작성하면 여러 문서에서 재사용할 수 있어 개발 시간을 단축하고 코드 중복을 줄입니다

Cascading의 의미

CSS는 Cascading Style Sheets의 약자입니다. 이는 Cascading한 속성을 가진 스타일 문서임을 나타냅니다.  
그렇다면  Cascading은 어떤 의미를 가지고 있을까요?
Cascading은 연속적인 이라는 뜻을 가지며 cascade는 폭포를 의미합니다. 
CSS에서 스타일이 적용될 때 우선순위를 가지고 적용되는데, 이 과정이 마치 폭포처럼 위에서 아래로 떨어지는 모양이기 때문에 Cascading이라는 단어가 붙었습니다.

기본 문법

선택자(Selector) { 속성(Property) : 속성값(Property Value) ; } 형태로 구성됩니다.

Can I Use

CSS 속성의 브라우저 호환성을 한눈에 확인할 수 있는 필수 도구입니다. 새로운 CSS 기능을 사용하기 전에 호환성을 항상 체크하는 습관을 들이는 것이 좋습니다.

핵심 개념

CSS의 Cascading은 스타일이 우선순위에 따라 위에서 아래로 떨어지는 폭포처럼 적용되는 것에서 유래된 이름입니다.

9.2 CSS 적용하기

1. 인라인 방식 (Inline Styles)

HTML 태그의 style 속성에 직접 스타일을 지정하는 방식입니다.

소규모 테스트에 유용하지만, 코드 가독성이 떨어지고 CSS의 가상 클래스나 가상 요소에 스타일을 줄 수 없어 일반적인 실무에서는 권장되지 않습니다.

2. 내부 스타일 (Internal Styles)

HTML 문서의 <head> 요소 안에서 <style> 태그를 사용하여 스타일을 정의하는 방식입니다.

3. 외부 스타일 (External Styles)

HTML 파일에 별도의 .css 확장자를 가진 외부 CSS 파일을 연결하는 방식입니다.

  • <link> 태그를 사용하여 <head> 요소 내부에 연결
  • rel: 외부 리소스와의 관계를 나타내며, 스타일 파일의 경우 stylesheet를 값으로 가짐
  • href: 참조할 CSS 파일의 위치를 나타냄

실무에서 가장 많이 사용되는 방법으로, HTML과 CSS를 분리하여 코드의 가독성, 효율성, 유지보수성을 높입니다.

4. 다중 스타일 시트

기본 (여러 <link> 태그 사용)

여러 link 태그를 사용하여 CSS 파일을 연결할 수 있으며, 나중에 작성된 스타일이 앞서 작성된 스타일을 덮어쓸 수 있으므로 순서가 중요합니다.

CSS 파일 안에 CSS 포함 (@import 구문)

CSS 파일 내에서 @import url('파일경로'); 구문을 사용하여 다른 CSS 파일들을 포함할 수 있습니다.

성능 측면: <link> 태그가 더 효율적입니다. <link>는 병렬 다운로드가 가능하지만, @import는 순차적으로 다운로드하여 렌더링 속도가 느려질 수 있습니다.

9.3 CSS 상속

상속(Inheritance)

부모 요소에 적용된 스타일 속성이 자식 요소에게 전달되는 메커니즘을 말합니다. 이는 HTML 문서의 계층 구조를 따라 스타일이 "흘러내리는" 방식으로 적용되어 전체적인 디자인의 일관성을 유지하고 코드 중복을 줄이는 데 도움이 됩니다.

상속되는 속성 vs 비상속 속성

  • 상속되는 속성: 일반적으로 텍스트 관련 속성들(color, font-family, font-size, text-align 등)
  • 비상속 속성: 레이아웃 관련 속성들(width, height, margin, padding, border, background 등)

우선순위

명시적으로 작성된 자식 요소의 속성값이 부모로부터 상속받은 속성값보다 우선순위가 높게 적용됩니다.

상속 제어 속성

  • inherit: 선택한 요소에 적용된 속성값을 부모와 동일하게 상속받게 합니다
  • initial: 요소의 모든 속성을 각 속성의 초기값으로 변경합니다
  • unset: 해당 속성이 자연적으로 상속되는 속성이면 inherit처럼, 아니면 initial처럼 작동합니다
  • revert: 스타일 규칙이 없었을 때 적용되었을 상태로 복구합니다

all 속성에 inherit, initial, unset, revert 값을 지정하여 모든 기존 스타일에 대한 상속 방식을 제어할 수 있습니다.

전체 선택자 (*)

HTML 문서 내의 모든 요소를 선택합니다.

  • 브라우저마다 다른 사용자 에이전트 스타일의 기본 스타일을 제거하는 데 주로 사용됩니다
  • 페이지 로드 성능에 영향을 미칠 수 있으므로 필요한 경우에만 사용하는 것이 좋습니다
  • 예: * { margin: 0; padding: 0; }

타입(유형) 선택자 (<태그명>)

특정 태그를 선택하여 스타일을 적용합니다 (예: h1, p).

  • 동일한 태그이면 모두 스타일이 적용되므로, 공통 스타일을 적용하는 데 사용됩니다
  • 세부 스타일에는 클래스나 아이디 선택자를 사용하는 것을 권장합니다

아이디 선택자 (#)

아이디를 사용하여 특정 요소를 선택합니다.

  • 아이디는 HTML 페이지 내에서 고유한 값을 가지므로, 한 개의 요소에만 스타일을 적용합니다
  • 재사용성이 떨어지며, 주로 자바스크립트 기능을 추가하거나 해시 링크 이동에 사용됩니다
  • 스타일링을 위해 id 선택자를 사용하지 않는 것이 좋습니다. CSS 우선순위가 높아 변경 및 덮어쓰기가 어렵습니다
  • 숫자로 시작할 수 없습니다 (예: #1id (X), #id1 (O))

클래스 선택자 (.)

클래스를 사용하여 요소를 선택합니다.

  • 하나의 요소에 여러 개의 클래스를 적용할 수 있으며, 스타일을 세분화하고 추가하는 데 유연합니다
  • 스타일 적용 시 가장 바람직한 방법으로 권장됩니다
  • 숫자로 시작할 수 없습니다 (예: .1class (X), .class1 (O))
  • 하이픈 (-), 언더바 (_), 문자로만 시작할 수 있습니다

속성 선택자 ([])

HTML 요소의 속성을 기반으로 요소를 선택합니다.

예: [속성이름], [속성이름="속성값"], [속성이름*="속성값"]

그룹 선택자 (,)

여러 개의 선택자를 쉼표(,)로 구분하여 한 번에 동일한 스타일을 적용합니다.

코드 중복을 줄이고 스타일 일관성을 유지하는 데 유용합니다.

10.2 복합 선택자

CSS 복합 선택자는 여러 선택자를 조합하여 더 구체적으로 요소를 선택합니다.

자손 선택자 (공백)

특정 요소의 모든 하위 요소를 선택합니다.

예: div p (div 요소 내의 모든 p 요소 선택)

자식 선택자 (>)

특정 요소의 직계 자식 요소만 선택합니다.

예: ul > li (ul 요소의 직계 자식인 li 요소만 선택)

일반 형제 선택자 (~)

선택자 뒤에 나오는 모든 형제 요소를 선택합니다.

예: p ~ span (p 요소 뒤에 나오는 모든 span 형제 요소 선택)

인접 형제 선택자 (+)

선택자 바로 뒤에 인접한 형제 요소만 선택합니다.

예: h1 + p (h1 요소 바로 뒤에 인접한 p 형제 요소 선택)

10.3 선택자 우선순위

여러 스타일이 충돌할 때 어떤 스타일을 적용할지 결정하는 규칙을 선택자 우선순위라고 합니다.

크게 세 가지 원칙을 따릅니다:

  1. 후자 우선의 원칙
  2. 구체성(명시도)의 원칙
  3. 중요성의 원칙

1. 후자 우선의 원칙

동일한 선택자에 동일한 속성이 여러 번 정의될 경우, 가장 마지막에 정의된 스타일이 적용됩니다.

예: p { color: blue; } p { color: red; } → 빨간색 적용

2. 구체성(명시도, Specificity)의 원칙

선택자가 얼마나 구체적으로 요소를 지정하는지를 나타냅니다. 더 구체적인 선택자가 우선순위가 높습니다.

가중치 계산 방법
  • inline-style: 1000점
  • id (#): 100점
  • class (.), 속성 선택자, 가상 클래스: 10점
  • 타입 선택자, 가상요소 선택자: 1점
  • 전체 선택자 (*): 0점

중요: 유형 선택자의 가중치는 클래스 선택자보다 우선될 수 없습니다. 클래스의 가중치를 먼저 비교하고, 다음으로 유형 선택자의 가중치를 비교합니다.

3. 중요성의 원칙 (!important)

!important 선언은 다른 모든 선언보다 최우선합니다.

매우 강력한 도구이지만 남용하면 CSS의 예측 가능성과 유지보수성을 해칠 수 있어 좋지 못한 습관으로 간주됩니다. 제한적인 상황 (외부 라이브러리 CSS 재정의, 유틸리티 클래스 강제 적용)에만 권장됩니다.

스타일의 우선순위 적용 순서

  1. 중요성의 원칙: !important가 있는 선언 우선
  2. 구체성의 원칙: 더 구체적인 선택자가 우선
  3. 후자 우선의 원칙: 같은 구체성을 가진 경우 나중에 선언된 스타일이 우선

10.4 네이밍 규칙

이름 짓기(네이밍)의 중요성

프로그래머들이 가장 힘들어하는 일 중 49%를 차지할 정도로 중요합니다.

  • 코드의 가독성을 높이고, 유지보수를 쉽게 만들며, 팀원 간의 협업을 원활하게 합니다
  • 프로젝트가 커질수록, 여러 개발자가 함께 작업할수록 체계적인 네이밍 규칙은 프로젝트 성공에 직접적인 영향을 미칩니다

1. 기본 네이밍 원칙

  • 의미 있는 이름 사용하기: 요소의 목적이나 내용을 명확하게 표현해야 합니다 (예: .nav-menu, .product-card)
  • 축약어 사용 주의: 과도한 축약은 가독성을 해치므로, 명확성을 위해 완전한 단어를 사용하되, 일반적으로 알려진 축약어(nav, btn)는 허용됩니다
  • 소문자 사용: CSS는 대소문자를 구분하므로, 일관성을 위해 소문자만 사용하는 것을 권장합니다
  • 특수문자 제한: 문자, 숫자, 하이픈(-), 언더스코어(_)만 사용 가능합니다
  • 숫자로 시작 금지: 클래스명은 숫자로 시작할 수 없으며, 반드시 문자와 하이픈, 언더스코어로 시작해야 합니다
  • 이름 길이 적절히 유지: 너무 길거나 복잡한 이름은 가독성을 해치고, 너무 짧고 모호한 이름도 피해야 합니다

2. 주요 네이밍 컨벤션

케밥 케이스 (kebab-case)
  • 단어 사이를 하이픈(-)으로 구분하는 방식입니다
  • CSS 클래스 네이밍에 가장 널리 사용됩니다
  • 장점: HTML과의 일관성, 가독성, 호환성 (모두 소문자), 국제화에 용이
카멜 케이스 (camelCase)

첫 단어는 소문자로 시작하고 이후 각 단어의 첫 글자를 대문자로 표기합니다. JavaScript에서 주로 사용하며, CSS에서는 권장되지 않습니다.

스네이크 케이스 (snake_case)

단어 사이를 언더스코어(_)로 구분하는 방식입니다. 파일명이나 데이터베이스 필드에 주로 사용되며, CSS에서는 자주 사용되지 않습니다.

3. 주요 네이밍 방법론

BEM (Block Element Modifier)
  • Block, Element, Modifier 세 가지 개념으로 구성됩니다
  • 클래스 이름만으로 HTML 구조를 이해할 수 있게 해줍니다
  • 예: .card (Block), .card__title (Element), .card--featured (Modifier)
  • 장점: 명확한 구조, 모듈성, 네임스페이스 충돌 방지, 쉬운 디버깅. 실무에서 널리 사용됩니다
OOCSS (Object Oriented CSS)

구조와 스킨을 분리하여 재사용 가능한 CSS를 작성하는 방법론입니다.

예: .btn (구조), .btn-primary (스킨). HTML에서 <button class="btn btn-primary">와 같이 두 클래스를 함께 사용합니다.

SMACSS (Scalable and Modular Architecture for CSS)

CSS를 Base, Layout, Module, State, Theme의 카테고리로 분류하여 구조화하는 방법론입니다.

코딩 컨벤션의 핵심

CSS 네이밍 규칙은 단순한 코딩 스타일 이상으로, 코드 품질, 유지보수성, 팀 협업에 중요합니다. 어떤 방법론을 선택하든, 일관성을 유지하고 의미를 명확하게 전달하는 것이 가장 중요합니다.

학습 목표

  • CSS 텍스트 속성들의 종류와 사용법 이해
  • 절대 단위와 상대 단위의 차이점과 활용 방법 학습
  • list-style 속성을 통한 목록 스타일링 기법 습득
  • 가상 클래스와 가상 요소의 개념과 실무 활용법 이해
  • CSS 변수와 :root를 활용한 효율적인 스타일 관리 방법 학습

11장 텍스트 관련 속성

11.1 텍스트 꾸미기

color (색상)

텍스트의 색상을 설정하며, font- 접두사가 붙지 않습니다.

표현 방법
  • 키워드: red, black 등 미리 정의된 값
  • Hex 코드: #으로 시작하는 6자리 16진수 (빨강, 초록, 파랑 강도)
  • RGB, RGBA: Red, Green, Blue 조합 (0~255). RGBA는 알파 값 (0~1)을 추가하여 투명도 설정
  • currentColor: 요소의 color 속성값을 나타냅니다

접근성: 저시력 시각 장애를 위해 폰트 색상과 배경의 명도 대비(최소 4.5:1)가 중요합니다.

font-family (글꼴)

글꼴을 지정하며, 여러 글꼴을 쉼표로 구분하여 작성합니다.

  • 왼쪽부터 순서대로 사용자의 컴퓨터에 설치된 글꼴을 찾아 적용합니다
  • 마지막에는 항상 일반 글꼴 계열(Generic Font Families)을 지정하는 것이 좋습니다
웹폰트 적용 방법
  • <link> 방식: HTML <head>에 직접 삽입. 폰트 로딩 시간 단축, CSS 파일 분리 관리 용이
  • @import 방식: CSS 파일 내 @import 구문 사용. <link> 방식보다 로딩이 늦을 수 있습니다
  • @font-face 방식: CSS에서 폰트 정의를 직접 작성. 여러 형식의 폰트 파일이 필요할 수 있습니다

font-size (글꼴 크기)

글꼴 크기를 조절합니다.

주요 단위
  • px: 절대 단위, 화면의 최소 단위인 픽셀 기준
  • em: 현재 요소의 부모 요소 글꼴 크기 기준의 상대 단위
  • rem: 루트 요소 (<html>) 글꼴 크기 기준의 상대 단위 ('root em')
실무 활용

반응형 웹페이지 개발 및 유지보수에 용이하여 em, rem과 같은 상대 단위를 주로 사용합니다. rem은 일반적으로 설계의 기반이 되고, em은 컴포넌트 내부 요소에 사용됩니다.

font-weight (글꼴 굵기)

텍스트의 굵기를 조절합니다.

  • 숫자 값: 100부터 900까지 100 단위로 지정 가능 (예: 400 (Normal), 700 (Bold))
  • 키워드: normal (400), bold (700), lighter, bolder
  • 주의: 모든 폰트가 9단계 굵기를 모두 지원하지는 않습니다

text-transform (텍스트 변형하기)

텍스트의 대소문자를 변형합니다 (영문 텍스트에 유용).

값: none, uppercase, lowercase, capitalize

text-decoration (텍스트 장식하기)

텍스트에 장식을 추가하는 단축 속성입니다.

  • 개별 속성: text-decoration-line, text-decoration-color, text-decoration-style, text-decoration-thickness
  • 활용: 링크 스타일링, 중요한 텍스트 강조, 삭제된 텍스트 표시

text-shadow (텍스트 그림자)

텍스트에 그림자 효과를 추가합니다.

값: offset-x (수평), offset-y (수직), blur-radius (흐림 정도, 선택적), color

text-align (텍스트 가로 정렬)

블록 레벨 요소 내부의 인라인 콘텐츠(텍스트, 인라인 요소)의 가로 정렬을 설정합니다.

값: left, right, center, justify

주의: justify (양쪽 정렬) 사용 시 흰 강 현상 (River of white)이 발생하여 가독성을 해칠 수 있으므로 권장하지 않습니다.

vertical-align (세로 정렬)

인라인/인라인 블록 요소나 테이블 셀 요소의 수직 정렬을 조정합니다.

  • 블록 레벨 요소에는 적용되지 않습니다 (블록 레벨의 수직 정렬은 Flexbox 사용)
  • 값: baseline (기본값), sub, super, top, text-top, bottom, text-bottom, middle

line-height (행간 조정)

텍스트 줄 사이의 간격(행간)을 설정합니다. 가독성에 큰 영향을 미칩니다.

값: normal (기본값), 숫자 (배수), 단위 (px, em, %)

팁: 본문 텍스트는 1.5 ~ 1.8 사이의 값을 사용하면 좋은 가독성을 얻을 수 있습니다.

기타 텍스트 관련 속성들

  • letter-spacing: 글자 사이의 간격(자간)을 조절합니다
  • text-indent: 텍스트가 시작하기 전 빈 공간(들여쓰기)을 설정합니다
  • word-break: 텍스트가 콘텐츠 박스 밖으로 넘칠 경우 줄바꿈 동작을 지정합니다
  • text-overflow: 부모 컨테이너를 넘어간 콘텐츠가 어떻게 보여질지 결정합니다

11.2 단위

웹 디자인에서 크기를 지정할 때 다양한 단위를 사용할 수 있습니다.

1. 절대 길이 단위

외부의 영향을 받지 않는 절대적인 값을 지니는 단위입니다.

  • px (픽셀): 디지털 화면의 이미지를 표현하는 가장 작은 단위
  • 고정된 값이라 직관적으로 이해하기 쉽고 사용하기 편리합니다

px 단위만 사용하면 안 되는 이유: 다양한 디바이스와 픽셀 밀도로 인해 디바이스별로 다른 위치, 크기로 보일 수 있습니다. 반응형 웹, 접근성, 사용자 경험을 위해 상대 단위 이해가 필수입니다.

2. 상대 길이 단위

외부의 영향(다른 요소나 뷰포트 크기 등)을 받아 상대적으로 크기가 변하는 단위입니다.

% (퍼센트)

부모 요소의 해당 속성 값을 기준으로 하는 상대 단위

뷰포트 단위 (vw, vh)

브라우저 창(뷰포트)의 크기를 기준으로 합니다. 1vw는 뷰포트 너비의 1%, 1vh는 뷰포트 높이의 1%

상대적 글꼴 크기 단위 (em, rem)
  • em: 부모 요소로부터 상속받은 글자 크기를 기준으로 하는 배수 단위
  • rem: 최상의 요소 (<html>)의 글자 크기를 기준으로 하는 배수 단위
em, rem 사용 상황
  • rem: 레이아웃 구성 요소, 텍스트 관련 속성. 전체 페이지에서 일관된 간격 유지
  • em: 컴포넌트 내부 요소 등 부모 요소의 크기에 비례하는 관계를 유지할 때

11.3 list-style

HTML의 순서 없는 목록 (<ul>)과 순서 있는 목록 (<ol>)의 스타일을 커스터마이즈합니다.

list-style-type

마커의 모양을 변경합니다.

주요 값: disc (채워진 원, 기본값), circle (빈 원), square (채워진 사각형), decimal (숫자), lower-alpha (소문자 알파벳), upper-roman (대문자 로마 숫자)

list-style-position

마커의 위치를 지정합니다.

  • outside: 리스트 항목 텍스트 블록 외부에 마커 배치 (기본값)
  • inside: 리스트 항목 텍스트 블록 내부에 마커 포함

list-style-image

기본 리스트 마커 대신 사용자 지정 이미지를 사용할 수 있게 해줍니다.

주의: 이미지 크기가 자동으로 조정되지 않으므로 적절한 크기의 이미지를 준비해야 합니다.

@counter-style

사용자 정의 카운터 스타일을 만들 수 있는 규칙입니다.

주요 속성: system, symbols, suffix, range, fallback

11.4 가상 클래스와 가상 요소

CSS에서는 특별한 상태나 위치를 지정할 수 있는 가상 클래스와 가상 요소를 제공하여 HTML 구조 변경 없이 세밀하고 동적인 스타일링이 가능합니다.

1. 가상 클래스 선택자 (Pseudo-class selectors)

요소의 특정 상태를 선택합니다. 콜론(:)을 사용하여 표현됩니다.

동적 가상 클래스 선택자
  • :link: 아직 방문하지 않은 링크
  • :visited: 이미 방문한 링크
  • :hover: 마우스 포인터가 올라간 요소
  • :active: 마우스로 클릭하고 있는 요소
  • :focus: 현재 초점을 가진 요소
구조적 가상 클래스 선택자
  • :first-child: 형제 요소 그룹 중 첫 번째 요소
  • :last-child: 형제 요소 그룹 중 마지막 요소
  • :nth-child(n): 부모 요소 내 모든 자식 중 지정된 순서의 요소 선택
  • :nth-of-type(n): 부모 요소 내 같은 타입의 형제 중 지정된 순서의 요소 선택
  • :not(selector): 특정 선택자를 제외한 요소를 선택
  • :has(selector): 특정 요소가 자식 요소를 포함하고 있는지를 검사하여 스타일을 적용

2. 가상 요소 (Pseudo-elements)

말 그대로 가상의 요소를 만들거나 선택한 요소의 특정 부분에 스타일을 적용합니다. 이중 콜론(::)을 사용하여 표현됩니다.

  • ::before, ::after: 선택한 요소의 내용 앞이나 뒤에 새로운 요소를 만들어 콘텐츠 추가
  • ::marker: 리스트 아이템의 마커를 스타일링
  • ::placeholder: <input>과 <textarea> 요소의 플레이스홀더 텍스트에 스타일을 적용

  • 장점: 스타일을 쉽게 재사용하고 관리할 수 있으며, 특히 브랜드 디자인 시스템 구축에 유용합니다
테마 전환

CSS 변수는 다크 모드와 라이트 모드 등 테마 전환에 특히 유용합니다. :root에 기본 테마 변수를 정의하고, JavaScript로 data-theme 속성만 변경하면 쉽게 테마를 전환할 수 있습니다.

12장 CSS의 기본, Box Model

12.1 CSS Box Model

CSS Box Model은 웹 페이지 레이아웃의 근간을 이루는 중요한 개념입니다.

모든 HTML 요소는 하나의 '상자'로 취급되며, 웹 페이지는 이런 상자들이 모여있는 것이라고 할 수 있습니다.

이 상자는 다음과 같이 구성됩니다:

  • 요소(element): 텍스트, 사진 등 실제 보여줄 대상
  • 패딩(padding): 요소 주변 영역을 감싸는 내부 여백
  • 테두리(border): 요소와 패딩을 감싸는 테두리
  • 마진(margin): 테두리 밖의 영역을 감싸는 외부 여백

주의사항: Box Model은 블록박스에 적용됩니다. 인라인 박스에는 width, height, 상하 margin 값이 적용되지 않습니다.

주요 속성들

width & height
  • width: 요소의 너비를 설정합니다 (기본값: auto)
  • height: 요소의 높이를 설정합니다 (기본값: auto)
padding (내부 여백)

콘텐츠 주변의 여백을 만듭니다 (element와 border 사이).

단축 속성으로, padding-top, padding-right, padding-bottom, padding-left 순서대로 작성합니다.

margin (외부 여백)

요소 주변의 여백을 만듭니다 (border 바깥).

요소의 배치 (가운데 정렬)
  • margin-left: auto;: 왼쪽 여백을 최대로 가져서 오른쪽 정렬
  • margin: 0 auto;: 좌우 여백을 자동으로 균등 분배하여 가운데 정렬
border (테두리)

요소의 테두리를 지정합니다. 선의 두께, 스타일, 색상 등 여러 속성의 단축 속성입니다.

box-sizing

요소의 총 너비와 높이를 계산하는 방식을 지정합니다.

  • content-box: 기본값. width = content 영역의 너비
  • border-box: width = content + border + padding

실무 권장: border-box를 사용하면 실제 렌더링되는 박스 크기를 직관적으로 제어할 수 있어 거의 표준처럼 사용됩니다. * { box-sizing: border-box; }로 모든 요소에 적용하는 것을 권장합니다.

기타 박스 관련 속성들
  • overflow: 요소의 콘텐츠가 요소의 크기를 초과할 때 어떻게 처리할지 지정
  • border-radius: 요소의 모서리를 둥글게 만듭니다
  • background: 요소의 배경을 설정하는 강력한 단축 속성
  • box-shadow: 요소에 그림자 효과를 추가
  • opacity: 요소의 불투명도를 설정 (0~1)

12.2 display 속성

display 속성은 CSS에서 레이아웃을 결정짓는 가장 중요한 속성 중 하나입니다.

요소가 문서 흐름에서 다른 박스들과 어떤 방식으로 배치될지를 결정합니다.

주요 display 값들

block
  • 요소 전후에 줄 바꿈을 생성
  • 새로운 줄에서 시작하며, 가능한 전체 너비를 차지
  • width, height, margin, padding 모두 사용 가능
inline
  • 요소 전후에 줄 바꿈을 생성하지 않음
  • 텍스트 흐름에 따라 같은 줄에 위치
  • width, height, 상하 margin 속성은 사용할 수 없음
inline-block

inline처럼 텍스트 흐름에 따라 배치되지만, block처럼 width, height, margin, padding을 모두 지정할 수 있습니다.

flex

내부 자식 요소들을 단방향(1차원적)으로 배치. 가로줄이나 세로줄을 만들어 요소들을 배치할 때 사용.

grid

내부 자식 요소들을 2차원적(격자 형태)으로 배치. 복잡한 레이아웃을 쉽게 구현.

none

해당 요소를 완전히 숨깁니다. 렌더링되지 않고, 레이아웃에서 공간을 차지하지 않으며, 접근성 트리에서도 제거됩니다.

웹 접근성을 고려한 숨김 처리

시각적으로 불필요하지만 보조도구 사용자에게는 필요한 정보를 숨기는 기술입니다.

  • display:none; or visibility:hidden;: 모든 사용자에게 콘텐츠를 숨깁니다
  • 콘텐츠를 화면 밖으로 보내기 (sr-only 클래스): 스크린 리더가 읽도록 합니다
  • CSS 클립 방법 (a11y-hidden 클래스): clip-path를 사용하여 스크린 리더가 읽지만 시각적으로는 숨깁니다

12.3 이미지 비율 유지하기

웹 디자인에서 이미지의 비율을 유지하는 것은 레이아웃의 일관성과 시각적 아름다움을 위해 중요합니다.

1. aspect-ratio 속성 사용하기

  • 요소의 기본 가로세로 비율을 설정합니다 (예: aspect-ratio: 2 / 1;)
  • 최신 브라우저에서 지원하며, 이미지 비율 유지를 위한 가장 직관적이고 간단한 방법입니다
  • object-fit 속성과 함께 사용하여 이미지가 설정된 비율을 유지하면서 요소를 채우도록 할 수 있습니다
object-fit

이미지나 비디오가 지정된 크기의 컨테이너에 어떻게 맞춰질지 결정합니다.

  • fill: 컨테이너를 가득 채우며, 원래 비율 변경될 수 있음 (기본값)
  • contain: 비율 유지하며 컨테이너 내 완전히 표시
  • cover: 비율 유지하며 컨테이너 가득 채움. 이미지 일부 잘릴 수 있음
  • none: 크기 조정 없이 원본 크기 그대로 표시
  • scale-down: none과 contain 중 더 작은 것을 적용

2. padding 백분율 값 활용하기

padding의 백분율 값은 항상 부모 요소의 너비를 기준으로 계산됩니다. 이 특성을 이용하여 화면 크기에 맞는 일정한 비율을 유지하는 박스를 만들 수 있습니다.

12.4 CSS 초기화

각 브라우저가 HTML 요소에 대해 기본적으로 제공하는 스타일(사용자 에이전트 스타일) 때문에 브라우저마다 동일한 HTML이 다르게 렌더링될 수 있습니다.

CSS 초기화(CSS Reset)는 이러한 브라우저 간 차이를 최소화하고, 개발자가 의도한 대로 스타일을 구현할 수 있는 일관된 기반을 제공하기 위해 사용됩니다.

크로스 브라우징(Cross Browsing)

웹 페이지가 다양한 브라우저와 디바이스에서 일관된 사용자 경험을 제공할 수 있도록 하는 웹 개발 방식입니다.

주요 CSS 초기화 방법

  • 에릭 마이어의 Reset CSS: 가장 널리 알려진 방식. 모든 요소의 마진, 패딩을 0으로 만들고, 기본 서식을 제거
    • meyerweb.com
  • Normalize.css: 브라우저 간 차이를 줄이되, 유용한 기본 스타일은 보존
    • normalize.css
  • sanitize.css: Normalize.css를 기반으로 하되, 더 많은 요소와 속성을 다루고, 현대적인 웹 개발 트렌드 반영
    • sanitize.css
  • CSS Remedy: 모던 CSS 기능을 활용한 최신 브라우저 대상 초기화 방식
    • [GitHub] cssremedy
  • 사용자 지정 Reset CSS: 팀의 특정 요구에 맞는 Reset CSS

13장 CSS 레이아웃 기초

13.1 position

position 속성: 문서상에 요소를 배치하는 방법을 지정하며, 복잡한 레이아웃 구성에 유용합니다.

주요 position 값

static (기본값)

HTML 작성 순서대로 정상적인 흐름(normal flow)에 따라 위치가 지정됩니다.

relative

요소가 원래 있어야 할 위치를 기준으로 상대적으로 위치를 조정합니다. 다른 콘텐츠들의 위치에 영향을 미치지 않으면서 자신의 위치만 조정할 수 있습니다.

absolute

가장 가까운 조상 중 position 속성이 static이 아닌 요소를 기준으로 절대적인 위치를 잡습니다. 일반적인 문서 흐름에서 제거되어 독립적으로 위치를 잡습니다.

fixed

뷰포트(브라우저 창)를 기준으로 고정 위치에 배치됩니다. 스크롤을 해도 화면의 같은 위치에 고정되며, 일반 문서 흐름에서 제거됩니다.

sticky

스크롤 위치에 따라 relative와 fixed 사이를 전환하는 특별한 속성값입니다. 스크롤되는 가장 가까운 상위요소를 기준으로 고정됩니다.

주의사항: position 속성을 사용할 때는 요소들이 서로 겹치거나 가려지지 않도록 주의하고, 브라우저 지원 여부를 확인하는 것이 중요합니다.

13.2 z-index

요소 쌓임 맥락 (Stacking Context)

웹 페이지에서 HTML 요소들이 3차원적으로 어떻게 배치되고 겹쳐질지를 결정하는 개념적인 공간입니다.

  • 웹 페이지의 최상위 요소인 HTML 태그에서 자동으로 생성됩니다
  • position 속성이 fixed나 sticky로 설정된 요소에서도 새로운 쌓임 맥락이 만들어집니다

z-index 속성

요소의 쌓임 순서(stacking order)를 결정하며, 어떤 요소가 다른 요소 위에 나타날지를 제어합니다.

z-index의 기본 특성

  • position이 static 외의 값을 가진 박스에만 z-index를 통한 Z축 순서 지정을 할 수 있습니다
  • 값이 클수록 제일 위로 배치됩니다
  • 부모가 z-index를 높여 자식 앞으로 나올 수는 없습니다
  • 자식은 z-index를 음수 값으로 주어 부모 뒤로 가는 것이 가능합니다
  • 부모 요소의 z-index가 낮으면 자식의 z-index가 아무리 높아도 한계가 있습니다
  • 유지보수를 위해 10이나 100 단위로 작업하는 것이 좋습니다

14.1 float

float 속성

요소를 페이지의 일반적인 레이아웃 흐름에서 벗어나게 하여, 텍스트나 다른 인라인 요소들이 그 주위를 감싸도록 만드는 레이아웃 기술입니다.

  • 값: left, right, none (기본값)
  • 주의: float를 적용하면 요소는 블록 레이아웃을 사용하게 됩니다

float 해제하기 (clear 속성)

float 사용 시 발생하는 레이아웃 문제를 방지하기 위해 clear 속성을 사용합니다.

  • clear: left: 왼쪽에 float된 요소 다음에 오도록 합니다
  • clear: right: 오른쪽에 float된 요소 다음에 오도록 합니다
  • clear: both: 모든 방향의 float를 한 번에 해제하는 가장 포괄적이고 안전한 방법으로 권장됩니다

float 사용 시 주의사항 해결

자식 요소들이 모두 float 속성을 가지면 부모 컨테이너의 높이가 자식 요소들을 포함하지 못하는 문제가 발생할 수 있습니다.

해결 방법
  1. 부모에 높이 값 지정: 유연성이 떨어집니다
  2. 부모에 overflow: hidden 적용
  3. clearfix 방법 사용: ::after 가상 요소를 사용하여 clear: both를 적용하는 가장 널리 사용되는 방법입니다

현대 웹에서의 활용: 최근에는 Flexbox나 Grid와 같은 현대적인 레이아웃 기술이 주로 사용되지만, float는 이미지를 텍스트 주변에 배치하는 등 특정 상황에서 여전히 유용하게 활용됩니다.

14.2 flex

Flexbox

현대 웹 디자인에서 가장 강력한 레이아웃 도구 중 하나로, 1차원적 레이아웃(행 또는 열)을 쉽게 만들고 요소들의 크기와 순서를 동적으로 조절할 수 있습니다.

flex 적용 방법: 부모 요소에 display: flex를 설정하면 자식 요소들이 flex item이 되어 flexbox 레이아웃을 사용할 수 있습니다.

flex-container에 사용하는 속성

flex-direction

주축의 방향을 설정합니다. (기본값: row)

값: row (왼쪽에서 오른쪽), column (위에서 아래), row-reverse, column-reverse

justify-content

주축을 기준으로 아이템들의 위치를 조절하거나 간격을 설정합니다.

값: flex-start, flex-end, center, space-between, space-around, space-evenly

align-items

교차 축을 기준으로 아이템을 정렬합니다.

값: stretch (기본값), flex-start, flex-end, center, baseline

기타 주요 속성들
  • align-content: 여러 줄의 flex item들을 교차축 방향으로 정렬
  • gap: 아이템 사이의 간격을 설정
  • flex-wrap: 한 줄에 배치되게 할 것인지, 여러 행으로 나누어 표현할 것인지 결정
  • flex-flow: flex-direction과 flex-wrap의 단축 속성

flex-item에 사용하는 속성

  • flex-basis: flex-item의 초기 크기를 설정 (기본값: auto)
  • flex-grow: 아이템이 flex container 내부에서 할당할 수 있는 공간의 비율을 지정
  • flex-shrink: 아이템의 크기를 축소할 때 사용
  • align-self: 개별 flex-item에 대해 교차축 정렬을 지정
  • order: flex-item의 배치 순서를 지정 (기본값: 0)
  • flex: flex-grow, flex-shrink, flex-basis의 단축 속성

14.3 grid

CSS Grid

flex가 1차원 레이아웃을 담당한다면, grid는 행과 열을 동시에 제어하여 2차원 레이아웃을 쉽게 만들 수 있는 기술입니다.

그리드의 기본 구성 요소

  • 그리드 컨테이너: 그리드의 가장 바깥 영역
  • 그리드 셀: 그리드의 한 칸 (개념적 정의)
  • 그리드 아이템: 그리드 컨테이너의 자식 요소
  • 그리드 트랙: 그리드의 행(row) 또는 열(column)
  • 그리드 라인: 그리드 셀을 구분하는 선
  • 그리드 갭 (gutter): 그리드 셀 사이의 간격
  • 그리드 에어리어: 그리드 셀의 집합

grid 적용 방법: 부모 요소에 display: grid를 설정하면 자식 요소들이 grid item이 되어 레이아웃을 사용할 수 있습니다.

grid-container에 사용하는 속성

행 방향과 열 방향 설정
  • grid-template-columns: 열 방향 그리드 트랙의 사이즈를 설정
  • grid-template-rows: 행 방향 그리드 트랙의 사이즈를 설정
  • fr 단위 (fraction): 컨테이너를 분할하여 트랙의 비율을 지정하는 유연한 길이 단위
함수
  • repeat(): row 또는 column 방향으로 grid-track의 사이즈를 간단하게 반복 설정
  • minmax(): 그리드에서 최소와 최대 사이의 범위를 설정
auto-fill과 auto-fit

repeat() 함수와 함께 사용되어 컨테이너의 너비에 따라 자동으로 그리드 컬럼을 배치합니다.

  • auto-fill: 가능한 많은 컬럼을 만들어내며, 컨테이너에 빈 공간이 남을 수 있습니다
  • auto-fit: auto-fill과 유사하나, 컨테이너 내부에 공간이 남을 경우 그 공간을 각 셀들이 나눠 갖습니다

그리드 아이템 속성

  • grid-area: 그리드 아이템의 위치와 크기를 지정하는 단축 속성
  • span: 행과 열을 병합할 때 사용 (예: span 3은 셀 세 개를 의미)
  • grid-template-areas: 영역을 정의하고 grid-area로 아이템을 배치
  • z-index: 그리드 내에서 아이템의 쌓임 순서를 제어 (position 속성 없이도 우선순위 설정 가능)

14.4 선택자 심화

고급 포커스 선택자

  • :focus-visible: focus와 유사하지만, 주로 키보드 탐색이나 스크립트 포커스 관리 시 활성화됩니다
  • :focus-within: 요소 또는 그 자식 요소가 포커스를 받았을 때 적용됩니다. 폼 디자인에 특히 유용합니다

상태 관련 가상 클래스 선택자

  • :enabled: 활성화된 상태의 폼 요소를 선택
  • :disabled: 비활성화된 상태의 폼 요소를 선택
  • :checked: 체크된 상태의 라디오 버튼이나 체크박스를 선택

속성 선택자 ([])

HTML 요소의 속성을 기반으로 요소를 선택하는 강력한 도구입니다.

  • [속성이름]: 해당 속성을 가진 모든 요소를 선택
  • [속성이름="속성값"]: 해당 속성에 해당 속성값을 가진 모든 요소를 선택
  • [속성이름^="속성값"]: 특정 문자열로 시작하는 요소를 모두 선택
  • [속성이름$="속성값"]: 특정 문자열로 끝나는 요소를 모두 선택
  • [속성이름*="속성값"]: 특정 문자열을 포함하는 요소를 모두 선택

15.1 transform

transform 속성

요소에 회전, 크기 조절, 기울이기, 이동 효과를 적용할 수 있게 해줍니다. 이 속성을 사용하면 요소의 형태와 위치를 동적으로 변경할 수 있습니다.

주요 transform 함수

translate()
  • 요소를 현재 위치에서 지정된 거리만큼 이동시킵니다
  • % 단위는 요소 자신의 크기를 기준으로 이동 거리를 계산합니다
  • 활용 팁: position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%); 코드로 요소를 정중앙에 정렬할 수 있습니다
scale()

요소의 크기를 조절합니다. 1보다 큰 값은 확대, 1보다 작은 값은 축소합니다. 요소의 내용물(텍스트 등)도 함께 확대/축소됩니다.

rotate()

요소를 지정된 각도만큼 회전시킵니다. 양수 값은 시계 방향, 음수 값은 반시계 방향으로 회전합니다.

skew()

요소를 X축이나 Y축을 기준으로 기울입니다. 과도한 skew() 적용은 텍스트의 가독성을 해칠 수 있습니다.

transform-origin

변형(transform)이 적용되는 기준점을 설정합니다. (기본값: 요소의 중심점 50% 50%)

translate() vs position (성능 및 사용 목적)

렌더링 과정 차이
  • position 변경 시: Layout 단계부터 다시 계산이 시작되어 성능상 불리할 수 있습니다
  • transform 변경: Composite 단계에서만 처리되어 성능상 이점을 가집니다
사용 지침
  • translate 권장 경우: 애니메이션이나 빈번한 위치 변경이 필요할 때, 성능이 중요한 상황
  • position 권장 경우: 정적인 레이아웃 구성 시, 복잡한 레이아웃 구조에서 요소를 배치할 때

15.2 transition

Transition

요소의 속성 값이 변할 때, 그 변화가 일정 시간에 걸쳐 점진적으로 일어나도록 하는 기능입니다.

주요 transition 속성

transition-property

전환 효과를 적용할 CSS 속성을 지정합니다. (all, none, 특정 속성 이름)

transition-duration

transition이 완료되는 데 걸리는 시간을 설정합니다. (초(s) 또는 밀리초(ms))

transition-timing-function

transition의 진행 속도를 설정합니다.

  • ease (기본값), linear, ease-in, ease-out, ease-in-out
  • cubic-bezier(n,n,n,n): 4개의 컨트롤 포인트를 사용해 transition의 시작부터 끝까지의 효과를 제어
  • steps(number_of_steps, direction): 변화의 단계를 설정. 프레임 기반 애니메이션이나 타이핑 효과 등에 유용
transition-delay

transition 시작을 지연시키는 시간을 설정합니다. (음수 값 사용 시 이미 진행 중인 것처럼 시작)

transition (단축 속성)

property, duration, timing-function, delay 순서로 모든 transition 관련 속성을 한 번에 설정할 수 있습니다.

주의: 모든 CSS 속성이 transition을 지원하는 것은 아닙니다. (예: display, outline 등)

15.3 animation

CSS Animation

여러 스타일을 전환시킬 수 있으며, JavaScript 없이 요소에 애니메이션 효과를 적용할 때 사용합니다. 애니메이션을 나타내는 CSS 스타일과 애니메이션의 중간 상태를 나타내는 키프레임들로 이루어집니다.

transition vs animation

  • transition: 요소의 상태가 변경되어야 애니메이션을 실행할 수 있습니다
  • animation: 요소의 상태 변화와 관계없이 애니메이션을 실행할 수 있습니다

@keyframes

두 개 이상의 애니메이션 중간 상태를 설정합니다.

  • from (0%)은 시작점, to (100%)는 종료 시점을 의미합니다
  • 중간 50% 등 여러 단계를 설정할 수 있습니다

주요 animation 속성

  • animation-name: @keyframes 규칙에 정의된 애니메이션의 이름을 지정
  • animation-duration: 애니메이션이 한 사이클을 완료하는 데 걸리는 시간을 지정
  • animation-timing-function: 애니메이션의 진행 속도를 제어하는 속성
  • animation-delay: 애니메이션 시작 시점을 지정 (양수: 지연 시작, 음수: 중간부터 시작)
  • animation-iteration-count: 애니메이션 재생 횟수를 설정 (infinite로 설정하면 무한 반복)
  • animation-direction: 애니메이션의 재생 방향을 지정 (normal, reverse, alternate, alternate-reverse)
  • animation-fill-mode: 애니메이션 실행 전과 후에 대상에 스타일을 적용하는 방법을 지정
  • animation-play-state: 애니메이션의 재생 상태를 설정 (paused, running)
  • animation (단축 속성): 모든 animation 관련 속성을 한 번에 설정

15.4 3D 관련 속성들

CSS 3D 속성

웹 요소를 3D 공간에서 변형하여 시각적으로 더 풍부하고 생동감 있게 만들 수 있습니다.

perspective

3D 위치 요소에 원근감을 주기 위해 z=0 평면과 사용자 사이의 거리를 결정합니다. 부모 요소에 적용하며 직계 자식 요소에만 영향을 미칩니다.

  • 값이 크면 변형이 작고, 값이 작을수록 변형이 커집니다
  • perspective는 부모 요소에 사용하는 반면, transform: perspective(800px)는 요소에 직접 적용합니다

perspective-origin

소실점의 위치를 지정합니다. 이는 사용자가 3D 공간을 바라보는 위치(x, y)를 결정합니다. (기본값: 정중앙 center)

transform (3D 변형과 관련된 메소드)

  • perspective() 함수: 3D 변형 메소드를 사용하기 위해 원근감을 표현할 기준을 명시하는 데 필수
  • translate3d(x, y, z): translate 속성의 3D 버전으로, x, y, z 축을 모두 한 번에 이동
  • rotate3d(x, y, z, angle): 해당 요소를 주어진 각도만큼 x, y, z 축을 기준으로 회전
  • scale3d(x, y, z): 해당 요소의 크기를 주어진 배율만큼 x, y, z 축 방향으로 늘리거나 줄임

기타 3D 관련 속성들

  • transform-style: 요소에 변형을 적용할 때 그 변환이 자식 요소에게도 적용될지를 설정 (flat, preserve-3d)
  • backface-visibility: 요소가 transform을 통해 3D 공간에서 회전되어 뒷면이 노출될 때, 그 뒷면을 보이게 할지 설정 (visible, hidden)

성능 고려: 복잡한 3D 변환은 렌더링 성능에 영향을 줄 수 있으므로, 필요한 경우에만 사용하고 가능한 간단하게 유지하는 것이 좋습니다.

  • 모바일 퍼스트 접근법과 브레이크포인트 설정 방법 학습

16장 반응형 웹 디자인

16.1 반응형 웹

반응형 웹 디자인

다양한 디바이스와 화면 크기에 맞춰 웹사이트의 레이아웃과 콘텐츠가 자동으로 조정되는 디자인 방식입니다.

주의: 특정 장치에 최적화된 별도의 URL(예: m.example.com)은 반응형 웹 디자인으로 간주하지 않으며, "적응형 웹 디자인" 또는 "별도의 모바일 사이트"로 분류됩니다.

반응형 웹의 필요성

데스크톱부터 스마트폰, 태블릿, 스마트 워치까지 다양한 기기로 웹사이트에 접속하는 현대 웹 환경에서, 하나의 웹사이트로 모든 기기에 대응하여 최적의 사용자 경험을 제공하기 위해 등장했습니다.

주요 장점

  • 사용자 경험 향상: 모든 디바이스에서 동일한 콘텐츠와 기능을 제공하여 사용자 경험의 일관성을 유지
  • 개발 및 유지 보수 효율성: 단일 코드베이스로 모든 플랫폼을 관리하여 개발 시간과 비용 절감
  • SEO 이점: Google 및 기타 검색 엔진이 모바일 친화적인 웹사이트를 선호
  • 미래 지향적: 새로운 디바이스가 출시되어도 기존 디자인이 대부분 적절히 대응 가능

반응형 웹의 핵심 기술

  • 유동적 그리드 (Fluid Grid): 픽셀(px) 대신 상대적인 단위(%, em, rem 등)를 사용하여 화면 크기에 따라 요소들의 크기가 자동으로 조절되도록 레이아웃을 구성
  • 유동적 이미지 (Flexible Images): 이미지도 컨테이너에 맞춰 크기가 조절되도록 설정 (img { max-width: 100%; height: auto; })
  • 미디어 쿼리 (Media Queries): 화면 크기에 따라 다른 CSS 스타일을 적용할 수 있게 해줍니다

반응형 웹 디자인 구현을 위한 핵심 팁

  • 모바일 퍼스트 접근 방식: 모바일 레이아웃부터 디자인을 시작하고, 점진적으로 큰 화면에 대응
  • 브레이크포인트 선택: 일반적인 디바이스 크기(예: 768px(태블릿), 1024px(데스크톱))를 고려
  • 콘텐츠 우선: 디바이스 크기에 관계없이 중요한 콘텐츠는 항상 접근 가능하도록
  • 성능 최적화: 이미지 최적화, 지연 로딩 등을 통해 모바일 환경에서도 빠른 로딩 속도를 유지

🎯 실습: Viewport 메타 태그 설정하기

HTML 문서의 <head> 섹션에 다음 메타 태그를 포함해야 합니다:

<meta name="viewport" content="width=device-width, initial-scale=1.0" />

  • width=device-width: 페이지 너비를 기기 화면 너비에 맞춥니다
  • initial-scale=1.0: 페이지가 처음 로드될 때의 초기 줌 레벨을 100%로 설정

실습 과제: 기존 HTML 파일을 열어 viewport 메타 태그가 올바르게 설정되어 있는지 확인하고, 없다면 추가해보세요.

16.2 CSS 함수

CSS 함수: CSS 속성 값으로 계산식을 지정할 수 있는 도구입니다.

주로 사용되는 함수

calc()

CSS 속성 값을 계산할 때 사용합니다. +, -, *, / 사칙연산이 가능하며, 서로 다른 단위끼리 계산할 수 있습니다.

주의: +와 - 연산자의 좌우 공백을 꼭 요구합니다.

min(), max()

여러 값 중 최소값 또는 최대값을 선택합니다.

clamp(최소값, 선호값, 최대값)

값을 지정된 범위 내로 고정합니다.

  • 선호값이 최소값과 최대값 사이에 있으면 선호값 사용
  • 선호값이 최소값보다 작으면 최소값 사용
  • 선호값이 최대값보다 크면 최대값 사용

🎯 실습: CSS 함수 활용하기

과제 1: calc() 함수를 사용하여 다음 CSS를 작성해보세요:

  • 전체 너비에서 200px를 뺀 크기: width: calc(100% - 200px);
  • 뷰포트 높이의 절반에서 헤더 높이(60px)를 뺀 크기: height: calc(50vh - 60px);

과제 2: clamp() 함수로 반응형 폰트 크기를 만들어보세요:

font-size: clamp(1rem, 2.5vw, 2rem);

이 코드는 최소 1rem, 최대 2rem 사이에서 화면 크기에 따라 폰트 크기가 조절됩니다.

16.3 미디어 쿼리

미디어 쿼리 (Media Query)

CSS3의 기능으로, 웹 페이지 또는 애플리케이션의 내용을 다양한 디스플레이 장치와 환경에 맞게 적응적으로 표시하도록 하는 방법을 뜻합니다.

미디어 유형 (Media Type)

스타일시트가 어떤 장치나 출력 방식에 적용되어야 하는지를 명시합니다.

주요 미디어 타입: all, screen, print, speech, projection, tv

미디어 쿼리 기본 구조

@media (미디어 특성) { /* 스타일 */ }

주요 미디어 특성

  • 너비 기반 미디어 쿼리: min-width (최소 너비 조건), max-width (최대 너비 조건)
  • 방향 기반 미디어 쿼리: orientation: portrait (세로 방향), orientation: landscape (가로 방향)
  • 화면 비율 기반 미디어 쿼리: aspect-ratio (예: 1/1, 16/9)
  • 해상도 기반 미디어 쿼리: resolution (출력 장치의 픽셀 밀도, 단위: dppx)

논리 연산자

  • and 연산자: 모든 조건을 만족하는 경우에만 스타일을 적용
  • not 연산자: 조건을 반전시킵니다
  • , (쉼표) 연산자: 나열된 조건 중 하나라도 만족하는 경우 스타일을 적용
  • only 키워드: 미디어 쿼리를 지원하는 브라우저에서만 스타일을 적용

브레이크 포인트 (Breakpoint)

반응형 웹 디자인에서 특정 화면 크기 또는 장치 특성에 따라 디자인이나 레이아웃이 변경되는 지점을 말합니다.

미디어 쿼리를 사용하여 min-width 및 max-width 등으로 브레이크 포인트를 정의하고, 해당 지점에서 적용될 스타일을 지정할 수 있습니다.

🎯 실습: 미디어 쿼리 작성하기

과제 1: 다음 조건에 맞는 미디어 쿼리를 작성해보세요:

  • 태블릿용 (768px 이상): @media (min-width: 768px) { }
  • 데스크톱용 (1024px 이상): @media (min-width: 1024px) { }
  • 가로 방향 화면: @media (orientation: landscape) { }

과제 2: 모바일 퍼스트 접근법으로 버튼 스타일을 작성해보세요:

/* 모바일 기본 스타일 */
.btn { font-size: 14px; padding: 8px 16px; }

/* 태블릿 이상 */
@media (min-width: 768px) {
  .btn { font-size: 16px; padding: 12px 24px; }
}

16.4 반응형 이미지

반응형 이미지

웹페이지는 다양한 디바이스 환경을 고려하여 적절한 이미지를 제공해야 합니다.

  • 해상도 전환 (Resolution Switching): 네트워크, 메모리, 성능을 고려하여 크기만 다른 동일 콘텐츠의 이미지를 보여주고 싶을 때 사용
  • 아트 디렉션 (Art Direction): 화면 크기, 방향 등을 고려하여 다양한 레이아웃의 이미지를 제공

해상도 전환 기법 (Resolution Switching)

srcset 속성

브라우저에 제시할 이미지 경로와 이미지의 원본 크기를 지정합니다.

  • w 서술자: 이미지의 원본 너비를 브라우저에게 알려줍니다
  • x 서술자: 화소의 밀도를 나타냅니다 (예: 1x, 2x)

주의: w 서술자와 x 서술자는 동시에 사용할 수 없습니다. srcset을 지원하지 않는 브라우저를 대비하여 src 속성은 필수로 입력해야 합니다.

sizes 속성

미디어(뷰포트) 조건문이 참인 경우, 이미지가 차지하게 될 사이즈를 브라우저에 알려줍니다. (px, em 또는 vw 단위 사용)

아트 디렉션 기법

<picture> 요소 사용

여러 <source> 요소와 하나의 <img> 요소를 포함합니다.

  • <source> 요소: 브라우저가 선택할 수 있는 이미지 소스를 제공. media와 type 속성을 가질 수 있습니다
  • <img> 요소: <picture> 요소 내에 필수적으로 포함되어야 하며, 대체 이미지로 사용됩니다

반응형 이미지 기법 정리

  • srcset과 sizes: 동일한 내용의 이미지를 다양한 디바이스의 적절한 해상도와 크기로 보여주고 싶을 때 사용 (브라우저가 알아서 이미지를 선택)
  • <picture>와 <source>: 브라우저 환경, 화면 크기, 방향 등을 고려하여 다양한 이미지를 보여주고 싶을 때 사용 (개발자가 더 세밀하게 이미지 선택을 제어)
  • 단순 스타일링 목적의 이미지: @media 미디어 쿼리를 사용하여 CSS의 background 속성으로 넣어주어도 됩니다

🎯 실습: 반응형 이미지 구현하기

과제 1: srcset을 사용한 반응형 이미지를 작성해보세요:

<img src="image-small.jpg" 
     srcset="image-small.jpg 400w, 
             image-medium.jpg 800w, 
             image-large.jpg 1200w"
     sizes="(max-width: 400px) 100vw, 
            (max-width: 800px) 50vw, 
            25vw"
     alt="반응형 이미지 예제">

과제 2: picture 요소로 아트 디렉션을 구현해보세요:

<picture>
  <source media="(min-width: 800px)" srcset="hero-desktop.jpg">
  <source media="(min-width: 400px)" srcset="hero-tablet.jpg">
  <img src="hero-mobile.jpg" alt="히어로 이미지">
</picture>
